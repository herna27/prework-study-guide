HTML

  • The body element represents the visible content shown to the user.
  • The body element represents the visible content shown to the user.
  • The footer element which is at the bottom of the webpage, usually contains the author, copyright, contact, sitemap and navigation.
  • The image element contains image for the webpage. The SRC attribute defines the images location and the ALT attribute describes the appearance and functionality of the image.
  • The br element creates a line of empty space or a line break between two blocks on content
  • The element that contains unordered or bulleted lists would be ul, ol, and li.
  • The p element represents a paragraph or block of text.
  • The level of heading element indicates the size of the heading text like for title and subtitle with h1 being the largest and each progressing number outputting a smaller size (h2, h3, h4, h5, h6).
  • Understanding the difference between tag vs element, as an element refers to the entire element and a tag whereas the tag is what appears inside the angle brackets.
  • Some common non-semantic elements are div which creates division in the webpage and span which creates an inline division.
  • The body element is an example of a semantic element as the name of the element describes it's contents.
  • Metadata is data about our webpage that the browser uses to effectively serve the webpage to users but typically not seen and is contained in the opening head element. Some metadata tags would include: title, style, base, link, meta, and script.
  • The head element includes nested elements that contain metadata that helps a browser render the page correctly. Some examples of this would be the document type declaration which is not case sensitive, the HTML opening tag which represents the root or start of our HTML document and also the language attribute which for English would be en.

CSS

  • CSS stands for cascading style sheets which is a programming language that defines how a webpage is presented to users whereas HTML strickly define the structure and content of a webpage.
  • CSS defines the colors, aesthetics, visual layout, and organization.
  • CSS defines style rules for specific elements of an HTML page and format them using a wide range of style options.

GIT

  • Git is a version control system that allows multiple people to collaborate on a project by tracking changes made.
  • Git commands can aid in tracking changes locally to your computer and used alongside with GitHub which makes collaborating with a team on a project much easier.
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely.
  • Git's branching and merging capabilities enable developers to experiment with new features or bug fixes without disrupting the main code.

JavaScript

  • JavaScript is a high-level programming language that allows users to interact with websites.
  • JavaScript can provide your website with effects, interactivity, data communication, and more.
  • JavaScript takes a HTML/CSS webpage and makes it interactive; for example, clicking a button, retrieving data, or filling out a form.
  • JavaScript is case sensitive so if you have a problem in the code check the case.
